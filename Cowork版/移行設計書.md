# 移行設計書：Cloud スケジュールタスク → Cowork スケジュールタスク + Computer Use

## 移行の背景

### 現行方式の問題

Cloud スケジュールタスクの `fetch` ツールでは、主要コンサルファーム・メディアのドメインが Bot/WAF 遮断により全て HTTP 403 で失敗した。

| ドメイン | 結果 | テスト日 |
|---|---|---|
| mckinsey.com | ❌ HTTP 403 | 2026-04-14 |
| bcg.com | ❌ HTTP 403 | 2026-04-14 |
| hbr.org | ❌ HTTP 403 | 2026-04-14 |
| ft.com | ❌ Claude Code側ブロック | 2026-04-14 |
| nikkei.com | ❌ HTTP 403 | 2026-04-14 |

成功率 0/5。詳細は [振り返り報告書.md](振り返り報告書.md) 参照。

### 新方式の選定理由

Cowork の Computer Use（Research Preview）を使えば、**ユーザーのPC上の実際の Chrome ブラウザ**を操作するため：

- User-Agent は通常の Chrome → Bot 判定されない
- ユーザーのログイン済みセッション（Cookie）をそのまま使える
- ペイウォール付きサイトも、事前ログイン済みなら記事本文を取得可能（の**見込み**）

> ⚠️ **「見込み」が実測で成立するかは [前提検証テスト手順書.md](前提検証テスト手順書.md) で確認する。前回の失敗は、まさに未検証の「見込み」を前提として構築したことが原因だった。**

## アーキテクチャ比較

### 現行（Cloud スケジュールタスク）

```
Anthropic Cloud → fetch（HTTP リクエスト）→ 403 ❌
```

### 新方式（Cowork スケジュールタスク + Computer Use）

```
ユーザーの Windows PC → Claude Desktop（Cowork）
  → ① Gmail コネクター（メール検索・取得）
  → ② Chrome ブラウザ操作（記事本文取得）
  → ③ ローカルファイル書き出し → git push
```

## Cowork の3層ツール優先順位（公式仕様）

> "In Cowork, Claude uses the most precise tool first."
> — https://support.claude.com/en/articles/14128542

| 優先度 | ツール | 用途 |
|---|---|---|
| 1（最優先） | **コネクター** | Gmail, Google Drive, Slack 等 → API 経由で最速・最安定 |
| 2 | **Chrome ブラウザ** | コネクターがないWebサービス → Chrome拡張経由でDOM操作 |
| 3（最終手段） | **Computer Use（画面操作）** | ブラウザでも対応できない場合 → スクリーンショット＋クリック |

本プロジェクトでは：

- **Gmail** → コネクター（第1層）で取得
- **記事本文** → Chrome ブラウザ操作（第2層）で取得
- **Computer Use（第3層）** → Chrome が対応できない場合のフォールバック

## 変更点サマリ

| 項目 | 現行 | 新方式 |
|---|---|---|
| 実行基盤 | Anthropic Cloud | ユーザーの Windows PC（Claude Desktop） |
| PC状態 | 不要 | **起動 & アプリ起動が必須** |
| メール取得 | Gmail MCP コネクター | Gmail コネクター（同じ） |
| 記事取得 | fetch → 403 失敗 | **Chrome ブラウザ操作 → 成功見込み（要実測）** |
| 認証 | 不可能 | **Chrome の既存ログインセッション流用** |
| 出力先 | GitHub push（claude/ ブランチ） | **ローカル + GitHub push** |
| スリープ対策 | 不要 | Settings > "Keep computer awake" ON |
| OS | 任意 | **Windows 10 以降**（Computer Use は Windows でも公式対応） |
| プラン | 任意の有料プラン | **Pro または Max** |

## 前提条件（新規）

- [ ] Claude Pro プラン
- [ ] Windows 10 以降（Computer Use 対応。Windows はトグル ON だけでセットアップ完了し、macOS のような権限付与は不要）
- [ ] Claude Desktop 最新版インストール済み
- [ ] Chrome ブラウザインストール済み
- [ ] Claude in Chrome 拡張機能インストール済み
- [ ] 対象サイトに Chrome でログイン済み
- [ ] PC のスリープ設定を無効化（または Settings > "Keep computer awake" ON）

## リスク評価

| リスク | 深刻度 | 緩和策 |
|---|---|---|
| **前提検証で NO-GO になる**（Chrome 経由でも取れない） | 🔴 高 | `前提検証テスト手順書.md` で最上流検証。NO-GO なら [振り返り報告書.md](振り返り報告書.md) §6 の代替アプローチへ |
| PC がスリープして実行されない | 🟡 中 | "Keep computer awake" ON。見逃し時は起動後に自動キャッチアップ（過去7日分の最新1回） |
| Computer Use が Research Preview で不安定 | 🟡 中 | Chrome ブラウザ操作（第2層）を優先し、Computer Use は最終手段に |
| スクリーンショットに機密情報が映る | 🟡 中 | 実行前に不要なアプリを閉じる |
| プロンプトインジェクション（Webページ経由） | 🔴 高 | 信頼できるサイトのみアクセス。CLAUDE.md の絶対遵守ルールを維持・強化 |
| Chrome のログインセッション切れ | 🟢 低 | 週次で Chrome のログイン状態を確認 |
| 権限プロンプトで停止 | 🟡 中 | 初回の Run now で Always allow を必ず選択 |
