# 📋 手順書：Cowork スケジュールタスク + Computer Use による「毎朝のトップファーム発信ダイジェスト生成」

## このドキュメントの位置づけ

本書は、**Claude Cowork のスケジュールタスク**（Desktop local task）と **Chrome ブラウザ操作 / Computer Use** を組み合わせて本リポジトリを運用するための**セットアップ & 運用手順書**。

前回版（Anthropic Cloud + fetch）は主要ドメインが HTTP 403 で全滅し、プロジェクトがクローズされた。本書はその再設計版であり、**記事取得を Chrome 実ブラウザ経由にすることで Bot 判定を回避**し、ユーザーのログイン済みセッションを活用する。

> ⚠️ **STEP 0 に進む前に必ず [前提検証テスト手順書.md](前提検証テスト手順書.md) を実施すること。** 同書が PASS しない限り、本書の STEP 0 以降に進んではならない。これは前回失敗の直接の再発防止である。

**各コンポーネントの詳細仕様は「実ファイル」と「設計書」を正とし、本書では埋め込みコードを持たない。**

## STEP 0：リポジトリの準備

### 0-1. 「タスクコード固め」方式の採用

Cowork スケジュールタスクも、指定したローカルフォルダ（working folder）のファイルを読み取れる。本リポジトリは**タスクの詳細指示を tasks/ templates/ config/ に固め、スケジュールタスクのプロンプト欄は「tasks/daily-email-scan.md を読んで実行せよ」という短い委譲だけにする**方式を採用している。

#### メリット

| メリット | 説明 |
|---|---|
| **タスク定義のバージョン管理** | Git で変更履歴が残る |
| **プロンプトの安定化** | スケジュールタスクのプロンプト欄は短く保ち、詳細はファイルに委譲 |
| **再利用・複製が容易** | 別のキーワードで同様のタスクを作りたい場合、ファイルをコピーして修正するだけ |
| **レビュー可能** | Pull Request でタスク定義の変更をレビューできる |

### 0-3. 各ファイルの役割

各ファイルの最新仕様は、冒頭の対応表から実ファイルを直接参照すること。

### 0-4. domain-access-levels.json の運用

ドメインごとに **Chrome ブラウザ操作によるアクセス可否**を分類：

| 分類 | 意味 | 運用時の扱い |
|---|---|---|
| 🟢 `full_access` | Chrome 経由で本文が取得できる | 通常通り要約 + 原文引用 + 和訳 |
| 🟡 `partial_access` | 冒頭/目次のみ取得可能 | 取得できた範囲で要約 + 「⚠️ 一部のみ取得可能」と注記 |
| 🔴 `no_access` | Chrome でも取れない（構造的ブロック） | 何も試みず、リンクのみ記載 + 「📌 手動確認」と明記 |

**初期値は空（未分類）**。前提検証テスト手順書の実測結果に基づいて初期化する。前回版では事前予想で `no_access` に埋めていたが、Cowork 移行で結果が変わる可能性があるため**必ず実測**すること。

## STEP 1：Claude Desktop のセットアップ

### 1-1. インストール

1. https://claude.com/download から最新版をダウンロード
2. インストール後、Claude Pro アカウントでログイン

### 1-2. Computer Use の有効化

1. Claude Desktop を開く
2. **Settings > General**（Desktop app セクション）を開く
3. **Computer use** トグルを **ON** にする

参照: https://support.claude.com/en/articles/14128542

### 1-3. Keep computer awake の設定

1. 同じく Settings > General
2. **Keep computer awake** トグルを **ON** にする

> ⚠️ ノートPCの蓋を閉じるとスリープするため、蓋は開けたままにする。
> 参照: https://code.claude.com/docs/en/desktop-scheduled-tasks

### 1-4. Claude in Chrome 拡張機能のインストール

記事取得の第一選択（Chrome ブラウザ操作）に必要。

1. Chrome ウェブストアから「Claude in Chrome」を検索してインストール
2. Claude Desktop と連携（拡張機能側のプロンプトに従う）
3. Chrome を再起動

### 1-5. 対象サイトへの事前ログイン

Chrome ブラウザで以下のサイトに**手動でログイン**しておく。Claude はこのログイン済みセッション（Cookie）を利用して記事を取得する。

| サイト | URL | 備考 |
|---|---|---|
| McKinsey | https://www.mckinsey.com/ | 無料登録で Insights 閲覧可 |
| BCG | https://www.bcg.com/ | 無料登録で Publications 閲覧可 |
| HBR | https://hbr.org/ | 有料購読推奨（月数本は無料） |
| Financial Times | https://www.ft.com/ | 有料購読必須 |
| 日経 | https://www.nikkei.com/ | 有料購読必須 |

> ⚠️ **Claude に認証情報（パスワード等）を直接渡さない。** Chrome で事前にログインしておけば、Claude はそのセッションを利用する。
>
> 公式: "Claude is trained to avoid... inputting sensitive data"
> — https://support.claude.com/en/articles/14128542

有料購読がないサイトは、前提検証テストの対象から除外してよい。代わりにレポートに「📌 手動確認」と記載される運用になる。

### 1-6. Gmail コネクターの設定

1. [claude.ai/settings/connectors](https://claude.ai/settings/connectors) にアクセス
2. **Google Workspace（Gmail）** を追加
3. OAuth 認証を完了

> **権限について**: Google の OAuth 画面では「メール送信」権限が表示されますが、これは Google 側の仕様です。Claude は読み取りとドラフト作成のみで、送信は行いません。
>
> 参照: [Use Google Workspace connectors](https://support.claude.com/en/articles/10166901)

## STEP 2：Cowork スケジュールタスクの作成

### 2-1. 方法A：Cowork UI から作成（推奨）

1. Claude Desktop を開き、左サイドバーの **「Scheduled」** をクリック
2. 右上の **「+ New task」** をクリック
3. **「New local task」** を選択
4. 以下を設定：

| 設定項目 | 入力内容 |
|---|---|
| **Task name** | `daily-consulting-digest` |
| **Description** | 毎朝のトップファーム発信ダイジェスト生成 |
| **Prompt** | 下記「プロンプト（コピー用）」を貼り付け |
| **Frequency** | **Daily** → 時刻ピッカーで **05:30** を選択 |
| **Model** | 利用可能な最新モデルを選択 |
| **Working folder** | `gmail-schedule` リポジトリのローカルパス（例: `~/repos/gmail-schedule`） |

5. **Save** をクリック

参照: https://support.claude.com/en/articles/13854387 / https://code.claude.com/docs/en/desktop-scheduled-tasks

### 2-2. 方法B：チャットから `/schedule` で作成

1. Cowork で新しいタスクを開く
2. チャット入力欄に `/schedule` と入力
3. Claude の質問に答えてタスクを設定

### 2-3. プロンプト（コピー用）

[CLAUDE.md](CLAUDE.md) の「スケジュールタスクのプロンプト（コピー用）」セクションを丸ごと使用する。プロンプト本体は本手順書には埋め込まない（二重管理防止）。

### 2-4. 初回の権限承認（重要）

タスク作成後、必ず **「Run now」で手動実行** し、表示される権限プロンプトで各ツール・アプリに対して **「Always allow」** を選択する。

> "To avoid stalls, click Run now after creating a task, watch for permission prompts, and select 'always allow' for each one."
> — https://code.claude.com/docs/en/desktop-scheduled-tasks

承認が必要になる対象：

- Gmail コネクター
- Claude in Chrome 拡張（Chrome ブラウザ操作）
- Computer Use（画面操作 / フォールバック用）
- ローカルファイル書き込み（reports/ insights/）
- git コマンド（commit, push）

**1つでも「Ask each time」のまま残すと、翌朝の自動実行が権限プロンプトで停止する。**

### 2-5. 実行時刻のずれ（公式仕様）

> "Each task gets a fixed delay of up to 10 minutes after the scheduled time to stagger API traffic. The delay is deterministic: the same task always starts at the same offset."
> — https://code.claude.com/docs/en/desktop-scheduled-tasks

5:30 設定の場合、実際の実行は 5:30〜5:40 の間になる。

### 2-6. 見逃し補填（キャッチアップ）

> "When the app starts or your computer wakes, Desktop checks whether each task missed any runs in the last seven days. If it did, Desktop starts exactly one catch-up run for the most recently missed time."
> — https://code.claude.com/docs/en/desktop-scheduled-tasks

PCがスリープしていた場合、起動後に最新1回分だけ自動実行される。

## STEP 3：動作確認

### 3-1. 事前：前提検証テスト（初回のみ）

[前提検証テスト手順書.md](前提検証テスト手順書.md) に従い、Chrome 経由での記事取得が実測で動作することを確認する。この結果で `config/domain-access-levels.json` を初期化する。

**本テストが PASS しない限り、STEP 3-2 以降に進んではならない。**

### 3-2. 手動テスト実行

1. 左サイドバーの **「Scheduled」** をクリック
2. `daily-consulting-digest` タスクを選択
3. **「Run now」** をクリック
4. **権限プロンプトが表示されたら「Always allow」を選択**（初回のみ）
5. 実行完了を待つ（10〜30分程度。Chrome 操作があるため前回版より長い）

### 3-3. 確認チェックリスト

| # | 確認項目 | 合格基準 |
|---|---|---|
| 1 | 日次レポートが `reports/` に生成されているか | ファイルが存在する |
| 2 | レポートが5層構造（Top Insights / Actions / Knowledge 8バケット / Raw Data / English Highlights）を満たすか | 各セクションが揃っている |
| 3 | セクション3の8プラクティス軸分類に `[C-suite: XX]` と `[業界: XX]` タグが付いているか | タグが全項目に付与されている |
| 4 | セクション4が `<details>` で折り畳まれていない（全文展開）か | 常時展開されている |
| 5 | セクション4にメール本文引用だけでなく**リンク先記事の原文引用 + 和訳**があるか | Chrome で取得成功した記事が引用されている。取得不可は URL + 📌手動確認 |
| 6 | セクション5の英単語表（10個）が存在し、例文がセクション4の引用から採られているか | 表が完成している |

| 9 | **実行者のメールアドレスが一切含まれていないか** | `[REDACTED]` または記載なし |
| 10 | 優先度Medium以上のインサイトが `insights/` に1ファイル単位で生成されているか | ファイルが存在する |
| 11 | インサイトYAMLに `practice` / `c_suite` / `sources` / `key_english_terms` が記入されているか | 全必須フィールドが存在する |
| 12 | インサイト本文に `## 📝 原文引用と和訳` セクションが存在し、英語原文が永続化されているか | セクションが存在する |

| 14 | `claude/` ブランチ（または main）にpushされているか | GitHubで確認 |
| 15 | Chrome ブラウザの権限が「Always allow」になっているか | 次回以降の自動実行で権限プロンプトが出ない |

### 3-4. 匿名化の追加テスト

テスト実行後、以下のコマンドでレポート・インサイトに本人情報が漏れていないか機械的に確認する：

```bash

grep -ri "@example.com" reports/ insights/
```

## トラブルシューティング

| 症状 | 原因の可能性 | 対処 |
|---|---|---|
| **タスクが実行されない** | PC がスリープ / Claude Desktop が閉じている | "Keep computer awake" ON を確認。蓋を閉じていないか確認 |
| **権限プロンプトで止まっている** | 初回実行時に「Always allow」を選択していない | Scheduled > タスク詳細 > Run now で再実行し、権限を承認 |
| **Chrome で記事が取得できない** | ログインセッション切れ | Chrome で対象サイトに手動で再ログイン |
| **Chrome 拡張が無反応** | Claude in Chrome 拡張の連携切れ | 拡張を再起動、Claude Desktop を再起動 |
| **「Computer use is in use by another session」** | 別の Claude セッションが画面操作中 | そのセッションを終了してから再実行 |
| **実行が非常に遅い（30分以上）** | Computer Use（第3層）に乱用フォールバックしている | Chrome 拡張が正しく動作しているか確認。Chrome ブラウザ操作（第2層）で済むはず |
| **レポートに本人名が漏れている** | LLMが匿名化ルールを無視した | タスク定義の匿名化ルールを強化し再テスト |
| **セクション3が旧3バケットで生成される** | タスク定義・テンプレートのキャッシュ | `tasks/daily-email-scan.md` と `templates/report-template.md` が最新であることを確認 |
| **セクション4が折り畳み（`<details>`）になっている** | 旧テンプレートが参照されている | `templates/report-template.md` が現行版か確認 |
| **セクション5の英単語表が生成されない** | LLMが5層目まで辿り着いていない | タスク定義の指示の明確さを確認 |
| **インサイトに原文引用がない** | 旧 insight-universal.md が参照されている | テンプレートが現行版か確認 |
| **リンク先が全て「取得不可」** | Chrome ログインセッション切れ / Bot対策強化 | 前提検証テストを再実行。Layer 3 が通らなければ NO-GO として一旦停止 |
| **git pushが失敗** | ローカルの git 認証 / ブランチ保護 | working folder で手動 push して原因を確認 |

```
│  ├── tasks/daily-email-scan.md                              │
│  ├── config/ (search-keywords, insight-config, ...)         │
│  ├── templates/                                             │
│  ├── docs/                                                  │
│  ├── reports/YYYY-MM-DD-*.md      ← 出力（自動生成）       │

└──────────────┬─────────────────────────────────┬────────────┘
               │ clone / pull                    │ push
               │                                 │
┌──────────────▼─────────────────────────────────▼────────────┐
│         ユーザーの Mac（Claude Desktop）                      │
│         Cowork スケジュールタスク 毎朝 05:30                  │
│                                                               │
│  プロンプト: tasks/daily-email-scan.md を読んで実行せよ       │
│         │                           │                         │
│         ▼                           ▼                         │
│  Gmail コネクター             Chrome ブラウザ操作             │
│  (read-only / 第1層)          (ログイン済みセッション / 第2層)│
│  メール検索・取得             記事本文の精読                  │
│                                     │                         │
│                              Computer Use（第3層・最終手段） │
│                              画面操作で取得                   │
└───────────────────────────────────────────────────────────────┘
```

| 項目 | 内容 |
|---|---|
| **実行方式** | Cowork スケジュールタスク（Desktop local task） |
| **実行環境** | ユーザーの Mac（Claude Desktop が起動している必要あり） |
| **実行時刻** | 05:30（最大10分のずれあり） |
| **監視対象** | `config/search-keywords.json` の `sender_emails` |
| **メール取得** | Gmail コネクター（第1層・最速） |
| **記事取得** | Chrome ブラウザ操作（第2層）→ Computer Use（第3層）→ URL記載のみ（フォールバック） |
| **認証** | Chrome の既存ログインセッションを利用（パスワード不要） |
| **分類軸** | プラクティス8バケット × C-suite × 業界 の3軸タグ付け |
| **副次成果物** | 日次 英単語10選 + インライン併記、インサイトDB |
| **匿名化** | 実行者の氏名・メアドはレポート/インサイトに一切記載しない |
| **出力先** | ローカル `reports/` `insights/` → GitHub push |
| **見逃し補填** | PC起動後に過去7日分の最新1回を自動キャッチアップ |
| **前提検証** | 月次で [前提検証テスト手順書.md](前提検証テスト手順書.md) を再実行 |
