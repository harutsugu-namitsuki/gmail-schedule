### 1-1. アカウント・プラン

- [ ] GitHub アカウントを持っている
- [ ] 対象リポジトリ（gmail-schedule）への push 権限がある

### 1-2. OS・アプリケーション

- [ ] macOS または Windows を使用している（Computer Use 対応OS）
- [ ] Claude Desktop 最新版をインストール（https://claude.com/download）
- [ ] Pro アカウントで Claude Desktop にログイン済み
- [ ] Chrome ブラウザがインストール済み
- [ ] Git がインストール済み & コマンドラインで使える

### 1-3. Claude Desktop の設定

- [ ] Settings > General > **Computer use** トグルを ON
- [ ] Settings > General > **Keep computer awake** トグルを ON
- [ ] **Claude in Chrome 拡張機能**をインストール済み（Chrome ウェブストアから）
- [ ] 拡張機能を Claude Desktop と連携済み
- [ ] Chrome を再起動した

### 1-4. 対象サイトへの事前ログイン（Chrome で手動実施）

以下は「このアカウントで有料購読している」か「無料アカウントでもログイン状態を作っている」サイトにのみチェック：

- [ ] McKinsey（https://www.mckinsey.com/）にログイン済み
- [ ] BCG（https://www.bcg.com/）にログイン済み
- [ ] HBR（https://hbr.org/）にログイン済み
- [ ] Financial Times（https://www.ft.com/）にログイン済み
- [ ] 日経（https://www.nikkei.com/）にログイン済み

> ⚠️ 有料購読がないサイトはチェック不要。前提検証の対象からも除外してよい。その場合、運用時は該当サイトが `no_access` に分類され、「📌 手動確認」として出力される。

## §3. 前提検証テスト（最重要ゲート）🚨

**§4 以降に進む前に、必ず本セクションを PASS させること。**

- [ ] [前提検証テスト手順書.md](前提検証テスト手順書.md) を読んだ

- [ ] Layer 1（トップページ）の結果を確認した
- [ ] Layer 2（個別記事URL）の結果を確認した
- [ ] Layer 3（実メール由来URL）の結果を確認した
- [ ] `config/domain-test-results.md` に結果が保存された
- [ ] GO / CONDITIONAL GO / NO-GO 判定を確認した

### 判定結果に応じた分岐

#### 🟢 GO の場合

- [ ] `config/domain-access-levels.json` を実測結果で初期化した（full_access / partial_access / no_access を記入）
- [ ] [前提検証テスト手順書.md](前提検証テスト手順書.md) §8「実施ログ」に GO 日付と合格根拠を記録した
- [ ] → §4 に進む

#### 🟡 CONDITIONAL GO の場合

- [ ] 取れるドメイン・取れないドメインを `domain-access-levels.json` に反映した
- [ ] 「情報量が十分か」を判断した（取れるドメインだけで運用する価値があるか）
- [ ] 判断結果を [前提検証テスト手順書.md](前提検証テスト手順書.md) §8 に記録した
- [ ] 運用価値があると判断した場合 → §4 に進む
- [ ] 運用価値がないと判断した場合 → §6「NO-GO 時の対応」に進む

#### 🔴 NO-GO の場合

- [ ] **§4 以降に進まない**
- [ ] [振り返り報告書.md](振り返り報告書.md) §6「代替アプローチ」を読み返す
- [ ] 次のアーキテクチャ案を検討する
- [ ] [前提検証テスト手順書.md](前提検証テスト手順書.md) §8 に NO-GO 日付・理由・次アクションを記録
- [ ] **本チェックリストはここで終了**。新アーキテクチャで別プロジェクトを起こす

## §4. スケジュールタスクの作成（§3 GO 後のみ）

### 4-1. タスク作成

- [ ] Claude Desktop の Scheduled サイドバーを開いた
- [ ] 「+ New task」→「New local task」を選択
- [ ] Task name: `daily-consulting-digest`
- [ ] Description: 毎朝のトップファーム発信ダイジェスト生成
- [ ] Frequency: Daily
- [ ] 時刻: 05:30
- [ ] Model: 最新モデルを選択
- [ ] Working folder: gmail-schedule のローカルパスを指定
- [ ] Prompt: [CLAUDE.md](CLAUDE.md) の「スケジュールタスクのプロンプト（コピー用）」を貼り付けた
- [ ] Save した

### 4-2. 初回権限承認（最重要）

- [ ] 作成したタスクを選択し、**Run now** をクリック
- [ ] 権限プロンプトが出たら **全て「Always allow」を選択**
  - [ ] Gmail コネクター
  - [ ] Claude in Chrome 拡張
  - [ ] Computer Use
  - [ ] ローカルファイル読み書き（reports/, insights/）
  - [ ] git コマンド（commit, push）

> ⚠️ 1つでも「Ask each time」のまま残すと、翌朝の自動実行が権限プロンプトで停止する（[失敗防止ガードレール.md](失敗防止ガードレール.md) G-5）。

## §6. NO-GO 時の対応（§3 で NO-GO だった場合のみ）

- [ ] 現プロジェクトをクローズ扱いにする
- [ ] 代替アプローチの選定会議を設定する
  - [ ] 候補A：RSS フィード方式
  - [ ] 候補B：メール本文のみ要約方式
  - [ ] 候補C：Firecrawl 等の専用スクレイピングサービス
  - [ ] 候補D：Substack 等 AI フレンドリーな情報源に絞る
- [ ] 選定結果を新規プロジェクトとして立ち上げる
- [ ] [振り返り報告書.md](振り返り報告書.md) に「Cowork 版も NO-GO に至った」事実を追記する
