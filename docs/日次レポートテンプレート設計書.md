# 日次レポートテンプレート（report-template）設計書

## 1. テンプレートの目的と位置づけ
本テンプレート（`report-template.md`）は、毎朝自動収集されるニュースやメール情報を、単なる「事実（What）の羅列」から、コンサルタントの意思決定とアクションを促す**「戦略的インテリジェンス（So What / Now What）」**へと昇華させるための出力フォーマットです。多忙なコンサルタントが「朝の30秒」で全体像を把握し、その日のクライアント対話等に直結させることを目的としています。

## 2. アーキテクチャと設計思想

本テンプレートは「トップダウン・アプローチ」と「ノイズ排除」の思想に基づき、以下の4層構造で設計されています。

### ① Executive Summary（サマリー層）
- **設計意図**: BLUF（Bottom Line Up Front：結論ファースト）の原則に従い、最も重要な情報を最上部に配置。
- **仕組み**: 抽出された全情報の中から、AIに「今日知っておくべきトップ3のトレンド」を選定させ、事実に加えて「それが我々のビジネスにどう影響するか（So What?）」という示唆を必ず併記させます。

### ② Actionable Recommendations（アクション層）
- **設計意図**: インテリジェンスを価値に変える「Now What?（次に何をすべきか）」を提示。
- **仕組み**: サマリーに基づき、コンサルタント（読者）が取るべき具体的な打ち手（例：特定クライアントへのアプローチ、特定競合の深掘り調査）の仮説を出力させ、思考のトリガー（壁打ち相手）として機能させます。

### ③ Strategic Categorization（テーマ別ハイライト層）
- **設計意図**: ピラミッドストラクチャーの第2階層。情報を時系列やランダムではなく、「戦うフィールド」ごとに構造化。
- **仕組み**: 「市場・競合」「顧客・対象業界」「技術・AI」など、戦略上重要なカテゴリーに情報を自動分類し、1件につき1〜2行で端的に記載させます。

### ④ Raw Data & Deep Dive（エビデンス層）
- **設計意図**: 一次情報へのアクセスを確保しつつ、普段は視界のノイズにならないように隠蔽（コンシーリング）する。
- **仕組み**: HTMLの `<details>` タグ（アコーディオン）を使用。普段は折り畳んでおき、気になったトピックのソースやメール原文を確認したい場合のみ展開して閲読できる設計。

## 3. アンチパターン（過去の課題と解決策）

| 過去の課題 (As-Is) | 本設計での解決策 (To-Be) |
|---|---|
| メールの内容を要約し、上から順にベタ打ちで並べていたため、ノイズが多い。 | 優先順位（Executive Summary）と戦略的カテゴリで情報を構造化。 |
| 最も重要な「総括」がレポートの一番下にあった。 | 情報のピラミッドを逆転させ、示唆とアクション案を最上段に配置。 |
| 「何が起きたか」しか分からず、提案に活かしにくい。 | 「インサイト（So What?）」「アクション（Now What?）」の記述を強制。 |

## 4. 運用上の留意事項
- 本テンプレートの出力を最大化するためには、AIに対するプロンプト側でも「カテゴリー分類のルール」「So Whatから出力させる指示」「優先順位付けアルゴリズム」などを厳密に指示する必要があります。
